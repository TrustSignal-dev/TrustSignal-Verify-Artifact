# Support

## Contact

For product support, Marketplace questions, or enterprise evaluation requests, contact:

- `support@trustsignal.dev`

For security issues, use the private reporting path in [SECURITY.md](SECURITY.md) instead of the support address.

## What To Include

When contacting support, include:

- repository or workflow name
- action version or full commit SHA in use
- relevant job logs with secrets redacted
- TrustSignal verification ID or receipt ID when available

## Response Guidance

TrustSignal support should acknowledge legitimate support requests promptly and keep security-sensitive discussions on the private security reporting channel.
